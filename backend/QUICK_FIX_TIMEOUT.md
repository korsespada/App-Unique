# Быстрое исправление таймаутов при фильтрации

## Проблема
При выборе категории + бренда одновременно запрос уходит в таймаут (60+ секунд).

## Причина
Отсутствуют композитные индексы для фильтрации по нескольким полям.

## Решение (5 минут)

### 1. Откройте PocketBase Admin
```
http://your-vps:8090/_/
```

### 2. Перейдите в Settings → Database

### 3. Выполните этот SQL:

```sql
CREATE INDEX IF NOT EXISTS `idx_products_status_brand_category` 
ON `products` (`status`, `brand`, `category`);

CREATE INDEX IF NOT EXISTS `idx_products_status_brand` 
ON `products` (`status`, `brand`);

CREATE INDEX IF NOT EXISTS `idx_products_status_category` 
ON `products` (`status`, `category`);
```

### 4. Нажмите Execute

### 5. Готово!

Теперь фильтрация по категории + бренду будет работать за 1-3 секунды вместо таймаута.

## Проверка

Выберите любую категорию и бренд в приложении - должно загрузиться быстро.

## Почему это работает?

Без индекса PocketBase сканирует всю таблицу products (может быть тысячи записей).
С индексом PocketBase сразу находит нужные записи через индекс (мгновенно).

Индекс работает как оглавление в книге - вместо того чтобы читать всю книгу, вы смотрите в оглавление и сразу находите нужную страницу.
